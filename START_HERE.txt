⚡ COMMON COMMANDS

  Start development (with hot reload):
  ─────────────────────────────────────
    docker-compose up


  Stop everything:
  ────────────────
    Ctrl+C (in PowerShell)
    or: docker-compose down (in another terminal)


  View logs while running:
  ────────────────────────
    docker-compose logs -f


  Rebuild after changing dependencies:
  ──────────────────────────────────────
    docker-compose up --build


  Start production mode:
  ──────────────────────
    docker-compose -f docker-compose.prod.yml up -d


  Check if containers are running:
  ──────────────────────────────────
    docker-compose ps


  Test the API:
  ─────────────
    curl http://localhost:8000/
    or visit: http://localhost:8000/docs

  More commands? See: QUICK_REFERENCE.txt

❓ FREQUENTLY ASKED QUESTIONS

  Q1: Do I need to change anything in these files?
  A:  No! Everything is preconfigured for your project.
      Just run: docker-compose up


  Q2: The container won't start. What do I do?
  A:  1. Press Ctrl+C to stop
      2. Run: docker-compose logs
      3. Check the error message
      4. See: USAGE_GUIDE.md → Troubleshooting


  Q3: I made changes to my code. Do I need to restart?
  A:  No! In development mode, changes auto-reload in ~1 second.
      (If not working, you might be in production mode)


  Q4: What's the difference between development and production?
  A:  Development (docker-compose.yml):
      • Auto-reload on code changes
      • Single process
      • Used for local development

      Production (docker-compose.prod.yml):
      • No auto-reload (not needed)
      • 4 concurrent workers (faster)
      • Auto-restart on failure
      • Used for real servers


  Q5: Can I test images with the API?
  A:  Yes! Visit: http://localhost:8000/docs
      Then use the /predict endpoint to upload images.


  Q6: How do I deploy to a server?
  A:  See: USAGE_GUIDE.md → Production Mode section


  Q7: What if port 8000 is already in use?
  A:  Run: docker-compose down
      Then: docker-compose up
      Or use: docker run -p 8001:8000 ... (different port)

🔑 KEY CONCEPTS

  • Container: A lightweight, isolated environment for your app
  • Docker: Tool that creates and runs containers
  • docker-compose: Simplified way to run containers (instead of long commands)
  • Dockerfile: Recipe that says how to build your container
  • Image: The built container (like a class)
  • Container: Running instance of an image (like an object)
  • Volume: A folder on your computer mounted inside the container
  • Port: Connection between your computer (8000) and container (8000)
